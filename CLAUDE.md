## Brand rules (non-negotiable)

- **Only accent:** #A63D1F (terracotta). No other accent colors.
- **All text:** #111111. Hierarchy via size and weight only, never color muting.
- **Buttons:** Pill-shaped only (border-radius: 999px). Never square.
- **No italic.** Ever.
- **Sentence case** always. `DL_` is the only exception.
- **Terracotta = punctuation, not decoration.** One per viewport, highest-signal element.

## Typography

Two fonts only:
- **Fira Code** (mono): DL_ mark, nav links, section labels `01 /`, tags `[Tag_]`
- **Inter** (sans): Everything else

## Component patterns

Import from `src/components/primitives/` for atoms.
Every component uses CSS variables from tokens вЂ” never hardcoded hex values.
All interactive components need `prefers-reduced-motion` support.
All components need `focus-visible` ring for accessibility.

## Metadata layer (agentic)

Every component has a JSON schema in `src/metadata/`. This is the layer AI agents read.

Schema fields:
- `id` вЂ” unique identifier
- `props` вЂ” typed prop definitions with defaults and descriptions
- `states` вЂ” all interactive states documented
- `tokens_used` вЂ” which design tokens this component consumes
- `anti_patterns` вЂ” what NOT to do (critical for AI correctness)
- `usage_guidelines` вЂ” when to use each variant
- `accessibility` вЂ” focus, contrast, motion requirements

Before generating any component, read its metadata file first.

## File conventions

- Components: PascalCase (`Button.jsx`)
- Tokens: kebab-case (`primitives.json`)
- Metadata: kebab-case (`button.json`)
- Stories: `ComponentName.stories.jsx`
- One component per file, default export

## When generating pages

1. Read component metadata from `src/metadata/` first
2. Import components from this library вЂ” never recreate them
3. Use Tailwind utility classes with token-backed values
4. Follow spacing tokens for all layout decisions
5. Check anti_patterns in metadata before rendering
6. Check brand rules checklist before considering done
